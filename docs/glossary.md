# Glossary and Acronyms

## Acronym Dictionary

| Acronym | Full Form | Context |
|---------|-----------|---------|
| **ACID** | Atomicity, Consistency, Isolation, Durability | Database transaction properties |
| **ADR** | Architecture Decision Record | Documented architectural decisions |
| **API** | Application Programming Interface | HTTP endpoints |
| **ARM64** | ARM 64-bit architecture | CPU architecture for Apple Silicon, AWS Graviton |
| **AMD64** | AMD 64-bit architecture | Standard x86-64 CPU architecture |
| **CI/CD** | Continuous Integration / Continuous Deployment | Automated pipeline |
| **CORS** | Cross-Origin Resource Sharing | Browser security policy |
| **CPU** | Central Processing Unit | Compute resource |
| **CQRS** | Command Query Responsibility Segregation | Architecture pattern |
| **CRUD** | Create, Read, Update, Delete | Standard data operations |
| **CSRF** | Cross-Site Request Forgery | Web security attack |
| **DB** | Database | Persistent data storage |
| **DDD** | Domain-Driven Design | Software design methodology |
| **Docker** | Containerization platform | Application packaging |
| **DTO** | Data Transfer Object | Data structure for API communication |
| **E2E** | End-to-End | Full system testing |
| **ER** | Entity Relationship | Database modeling |
| **ESLint** | JavaScript linter | Code quality tool |
| **ESM** | ES Modules | JavaScript module system |
| **GHA** | GitHub Actions | CI/CD platform |
| **HTTP** | Hypertext Transfer Protocol | Web communication protocol |
| **HTTPS** | HTTP Secure | Encrypted HTTP |
| **JWT** | JSON Web Token | Authentication token format |
| **K8s** | Kubernetes | Container orchestration |
| **LB** | Load Balancer | Traffic distribution |
| **MFA** | Multi-Factor Authentication | Additional auth factor |
| **MITM** | Man-In-The-Middle | Network attack |
| **MVC** | Model-View-Controller | Architecture pattern |
| **OOM** | Out of Memory | Process termination due to memory exhaustion |
| **ORM** | Object-Relational Mapping | Database abstraction layer |
| **OWASP** | Open Web Application Security Project | Web security standards |
| **P1/P2/P3** | Priority levels (Critical/High/Low) | Incident severity |
| **PK** | Primary Key | Unique row identifier |
| **REST** | Representational State Transfer | API architectural style |
| **RBAC** | Role-Based Access Control | Authorization model |
| **SHA** | Secure Hash Algorithm | Git commit identifier |
| **SQL** | Structured Query Language | Database query language |
| **SSL** | Secure Sockets Layer (deprecated, now TLS) | Encryption protocol |
| **TLS** | Transport Layer Security | Encryption protocol |
| **XSS** | Cross-Site Scripting | Web security attack |
| **Zod** | Schema validation library | Input validation |

## Glossary

| Term | Definition |
|------|-----------|
| **Arcjet** | Cloud-based security platform providing bot detection, rate limiting, and attack shield protection |
| **bcrypt** | Password hashing algorithm with adaptive cost factor and built-in salt |
| **Clean Architecture** | Software architecture pattern emphasizing dependency inversion and domain-centric design |
| **Connection Pooling** | Technique to reuse database connections instead of creating new ones per request |
| **Cookie** | Small piece of data stored by the browser, sent with every request to the origin |
| **Drizzle** | Lightweight ORM for TypeScript/JavaScript with SQL-like API |
| **Defense in Depth** | Security strategy using multiple layers of protection |
| **Docker Compose** | Tool for defining and running multi-container Docker applications |
| **Docker Multi-Stage** | Dockerfile pattern using multiple FROM statements for optimized builds |
| **Graceful Shutdown** | Handling process termination signals to complete in-flight work and clean up resources |
| **Helmet** | Express middleware that sets secure HTTP headers |
| **Hexagonal Architecture** | Architecture pattern with ports and adapters for external communication |
| **HTTPOnly Cookie** | Cookie flag preventing JavaScript access (XSS protection) |
| **Import Maps** | Node.js feature allowing path aliases in imports (via package.json imports field) |
| **Layered Architecture** | Software design separating concerns into stacked layers |
| **Neon Local** | Ephemeral PostgreSQL proxy for local development (emulates Neon Cloud) |
| **Neon Serverless** | Serverless PostgreSQL platform with database branching and auto-scaling |
| **Onion Architecture** | Architecture pattern with concentric layers of dependency |
| **Rate Limiting** | Technique to control the rate of requests from a client |
| **SameSite** | Cookie attribute controlling cross-site request behavior (CSRF protection) |
| **Serverless** | Cloud computing model where the provider manages infrastructure |
| **Sliding Window** | Rate limiting algorithm using a moving time window |
| **Subpath Imports** | Node.js feature for creating import aliases (package.json imports) |
| **Winston** | Multi-transport asynchronous logging library for Node.js |
